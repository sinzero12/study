## HCCEPose
***
#### BOP
- HCCEPose 코드 - BOP 형식 따르게 되어있음   

- 표준 데이터셋 : 사진들에 있는 물체가 무엇인지 맞추기 위한 이미지 제공
- 표준 포맷(BOP Format) : 폴더명(`train`, `test`, `models`), 파일명(`scene_gt.json`, `camera.json`), 좌표계 규칙 정함
- 표준 채점기 : AI 예측 위치-실제 정답 위치 얼마나 차이나는지 계산(평가)해주는 `bop_toolkit` 제공
***
#### 환경설정
- 다운로드
    - `git clone` 코드 다운
- 프로그램 설치
    - `apt-get` 리눅스 시스템 도구 설치
    - `pip install` 파이썬 라이브러리 설치

#### 데이터 준비
- 객체 전처리 (3D 파일 형식 바꾸고 중심 맞추기)
    - 3D 파일(STL) -> 형식(PLY)로 바꾸기

- 대칭 확인 (KASAL 이용)
    - 회전했을 때도, 똑같이 생긴 모양인지 확인

- 학습용 가짜 사진 만들기 (Rendering PBR)
    - 3D 모델 이용해, 컴퓨터가 가짜 사진(PBR) 수만장 만들어내는 것. 이때 사용하는 도구가 `BlenderProc`

#### 2D 탐지기 학습 (3D 자세 알기 전, 사진에서 물체가 어디에 있는지 네모 박스로 찾는 단계)
- YOLOv11 사용
- 데이터 변환(PBR 데이터 -> YOLO가 공부할 수 있는 형식으로 바꿔주기)

#### HCCEPose 모델 학습
- 물체 앞/뒷면 좌표 만들기
- 학습 
    - 모델 불러오기 : `HccePose.tester` - 2D 객체 탐지, 분할 및 6D 자세 추정 포함한 통합 테스터 모듈
                    , `HccePose.bop_loader` - 객체 모델 및 학습 데이터 로드 위한 BOP 형식 데이터셋 로더
